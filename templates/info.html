📘 NLP Pipeline

How Summarify Works

A deep-dive into the NLP algorithms powering real-time summarization

📥

1. Input Collection

Raw text is accepted via direct paste, .txt file upload, or .pdf extraction using PyPDF2. Multiple documents are concatenated before processing.

→
🔬

2. NLP Preprocessing

The text is sentence-tokenized using NLTK's Punkt tokenizer. Stopwords are removed. Each sentence is converted to a TF-IDF sparse vector for similarity computation.

nltk · sent_tokenize · TF-IDF
→
🕸️

3. Extractive Summarization (LexRank)

A sentence similarity graph is built where edge weights are cosine similarities. LexRank power-iteration (similar to PageRank) scores each sentence. The highest-scoring sentences—in original order—form the extractive summary.

LexRank · cosine similarity · graph ranking
→
🤖

4. Abstractive Summarization (T5)

The T5-small transformer model (Hugging Face) generates a human-readable paraphrase. The model uses seq2seq attention to rewrite content rather than extract it. Input is truncated to ~400 words for optimal performance.

transformers · T5-small · pipeline("summarization")
→
✨

5. Output Generation

Five output formats are generated simultaneously: Paragraph, Bullet Points, Headline, Gist, and Action Points. Important sentences are highlighted in the original text using <mark> tags.

→
📊

6. Analytics

Compression ratio, word counts, estimated reading times, and sentences saved are computed and displayed as analytics cards for transparency and insight.

Core Technologies

🔤

Tokenization

NLTK's Punkt sentence tokenizer splits text at sentence boundaries using an unsupervised algorithm trained on abbreviations and collocations.

📐

TF-IDF Vectors

Term Frequency–Inverse Document Frequency weighs words by their importance within and across sentences, enabling meaningful cosine similarity.

🕸️

LexRank Algorithm

Graph-based sentence scoring using eigenvector centrality. Sentences central to the document's discourse score highest. Implemented with power-iteration (50 steps, damping 0.85).

🧠

T5 Transformer

Google's Text-To-Text Transfer Transformer converts every NLP task to a text generation problem. T5-small has 60M parameters and generates fluent abstractive summaries.

📏

Cosine Similarity

Measures angle between TF-IDF vectors. Two sentences with similar vocabulary score high. Used to build the LexRank similarity matrix.

⚡

Action Detection

Rule-based keyword matching (should, must, required, need to…) extracts actionable sentences—critical for meeting notes and task-oriented documents.
